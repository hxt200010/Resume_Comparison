Build a full-stack ATS-style resume screening web application with a modern, professional UI.

PROJECT GOAL:
Create a website that simulates how companies use an Applicant Tracking System (ATS) to screen resumes against a job description. The app should help determine whether a resume is relevant for a job, estimate match quality, highlight missing skills, and explain why a candidate is a fit or not a fit.

IMPORTANT:
Do not make this a fake keyword counter only. I want a stronger ATS-style evaluator that combines:
1. keyword and phrase matching
2. semantic similarity
3. required skill matching
4. preferred skill matching
5. experience relevance
6. education relevance
7. project relevance
8. resume section parsing
9. filtering / reject logic for missing must-have requirements

The goal is to make the matching logic as realistic and accurate as possible for a student project, while being transparent that it is a simulation and not a real enterprise ATS.

MATCHING / ATS LOGIC:
Implement a realistic scoring pipeline with weighted scoring.

Suggested scoring categories:
- Required Skills Match: 30%
- Preferred Skills Match: 15%
- Semantic Similarity between resume and job description: 20%
- Relevant Experience Match: 15%
- Projects Relevance: 10%
- Education / Certifications: 5%
- Resume Completeness / Formatting Quality: 5%

Required filtering logic:
- If too many must-have skills are missing, mark as Reject / Not Relevant
- If minimum experience required is not met, lower score significantly
- If the resume domain is clearly unrelated to the job, lower semantic score and mark low relevance
- If job requires a degree/certification and it is missing, flag it clearly
- If core technologies are missing, explain that candidate may not pass ATS screening

Resume parsing requirements:
- Extract text from PDF and DOCX
- Attempt to detect sections using regex + NLP
- Normalize skills and synonyms
- Example: “ML” = “Machine Learning”, “JS” = “JavaScript”
- Detect years of experience when possible
- Detect named tools / technologies / frameworks
- Detect action-oriented project descriptions

NLP requirements:
- Use sentence-transformers to compute semantic similarity between:
  - full resume vs full job description
  - experience bullets vs job description
  - project bullets vs job description
- Use a predefined and extendable skill ontology / dictionary for tech roles, especially:
  - software engineering
  - machine learning
  - data science
  - cloud / DevOps
- Include fuzzy matching for related skill terms

RESULTS EXPLANATION:
The output should not just give a score. It should explain:
- why the resume matched
- which parts were relevant
- which must-have qualifications were missing
- whether the candidate would likely pass an initial ATS screen
- what improvements would increase the score
